# Raw Capture Logs

This directory contains raw LLM request/response captures collected by `llm-inspector` during testing sessions.

## What Is Captured

Each capture file is a JSON object with this structure:

```json
{
  "timestamp": "2026-04-11T00:45:03.632Z",
  "method": "POST",
  "path": "/v1/messages",
  "url": "/v1/messages?blthaiku=1",
  "headers": { "anthropic-version": "...", "x-api-key": "[REDACTED]" },
  "body": { ... },       // full request body (see below)
  "bodySize": 175405,
  "response": {          // present on non-streaming or post-stream capture
    "status": 200,
    "body": { ... },
    "usage": { "input_tokens": 1234, "output_tokens": 567 }
  }
}
```

### Request Body Fields

| Field | Description |
|---|---|
| `model` | Model ID (e.g. `claude-haiku-4-5-20251001`) |
| `messages` | Array of `{ role, content }` message objects |
| `system` | System prompt string (can be large) |
| `tools` | Array of tool definitions (often the biggest cost) |
| `metadata` | Request metadata (session info, etc.) |
| `max_tokens` | Max output tokens |
| `thinking` | Thinking budget config (if enabled) |
| `stream` | Whether response is streamed |

### Key Observations

- **Most captures are trivial test prompts** ("hello", "list files") — likely from proxy manual testing.
- The one substantive capture is `capture-2026-04-11T00-45-03-632Z-1775868307951.json` — a full `claude-haiku` session with 52 tools and a 28 KB system prompt.
- Token usage was only recorded on a few captures (response was mostly streaming, which captures the response body differently).
- The large haiku capture confirms the context overhead breakdown shown in the README: built-in tool schemas (~92 KB), system prompt (~28 KB), MCP tool schemas (~27 KB) are the dominant costs.
